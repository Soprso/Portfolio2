Understanding Dependency Injection in .NET

Dependency Injection (DI) is one of the most important design patterns in modern .NET development. It allows you to write cleaner, more testable, and maintainable code by decoupling object creation from object usage.

üîπ Why Use Dependency Injection?

  • Reduces tight coupling between classes.
  • Makes unit testing easier by allowing mock dependencies.
  • Improves code maintainability and scalability.
  • Promotes the SOLID principles, especially the Dependency Inversion Principle.

üîπ Basic Example in .NET

In .NET Core, DI is built into the framework. Here‚Äôs a simple example:


public interface IMessageService
{
    void Send(string message);
}

public class EmailMessageService : IMessageService
{
    public void Send(string message)
    {
        Console.WriteLine($"Email sent: {message}");
    }
}

public class NotificationController
{
    private readonly IMessageService _messageService;

    public NotificationController(IMessageService messageService)
    {
        _messageService = messageService;
    }

    public void Notify(string message)
    {
        _messageService.Send(message);
    }
}

Registering Services in Program.cs


var builder = WebApplication.CreateBuilder(args);

// Register dependency
builder.Services.AddScoped<IMessageService, EmailMessageService>();

var app = builder.Build();
app.Run();

üîπ Best Practices

  • ‚úÖ Prefer constructor injection over property injection.
  • ‚úÖ Keep registration organized (e.g., in extension methods).
  • ‚úÖ Avoid service locator patterns; let the framework handle injection.
  • ‚úÖ Use the correct lifetime (Scoped, Singleton, Transient) for each service.
